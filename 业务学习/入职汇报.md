# 入职汇报提纲

## 工作能力成长

1. 为什么做，理解了业务才能做对事
2. 如何做好，在所有的优化和测试工作中精准执行是基础
3. 和谁一起做，需要整合资源积极沟通去实现项目目标
4. 以什么心态做，心态从被动接收完成任务转变成主动担当

## 收获与心得体会

业务是根基： 制造业IT的核心价值在于解决实际业务问题。深入理解钢铁生产、物流、质量、成本等业务流程是开发出好用系统的前提。不懂业务，代码只是空中楼阁。(举例：扣重流程重构、复检断点消除)

细节决定成败： 接口的一个字段、页面的一次赋值、履历的一条信息，都可能成为线上问题的源头或用户体验的痛点。严谨细致是制造业IT人的必备素质

沟通协作是生命线： 大型集群项目涉及多系统、多团队、多角色用户之间的合作。清晰表达、积极倾听、换位思考、有效协作是项目顺利推进的保障。

拥抱变化，持续学习： 业务需求会变，技术也在更新。保持学习心态，快速适应变化，主动学习新知识 (如可视化技术、更深入的业务知识) 是持续成长的关键。

从“做任务”到“担责任”： 心态转变：从完成指派任务，到主动思考如何做得更好、如何解决问题、如何为项目和团队创造价值。

理论与实践的结合： 培训的知识只有在项目中实践才能真正内化。项目是最好、最快的成长课堂。

## 给新人的建议

1. 主动学习，不懂就问： 部门里有丰富的资源和乐于助人的同事，别害怕开口问问题，迈出了第一步以后，问得越多，后期成长越快。业务、技术、流程都要问。

2. 深入业务场景： 进入项目后多和用户交流，理解他们为什么需要这个功能，痛点在哪里。这会让你开发的东西更有价值。

3. 注重基础与规范： 编码规范、文档编写、测试用例，这些看似枯燥的基础工作，是保证质量和效率的基石。写好注释，当有一个几个月前的需求需要进行修改时能减少重新理解的时间

4. 培养沟通习惯： 及时同步进度，清晰反馈问题。良好的沟通能事半功倍。
